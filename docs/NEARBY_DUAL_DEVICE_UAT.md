# PairNest 双机 Nearby 真机验收手册

版本：`0.1.0`  
更新时间：`2026-05-26`

目标：验证“靠近自动同步、无服务端、本地优先、强私密性”在双机上可成立。

## 1. 测试准备

设备与环境：

- Android 真机 A、Android 真机 B（建议 Android 10+）
- 两台设备安装同一构建版本
- 蓝牙、定位、WiFi 打开
- 两台设备靠近（1~2 米内）

应用前置：

- 两机首次启动后完成权限授权（相机、相册/存储、Nearby 相关权限）
- 确保两机使用同一 `pair_id`（通过扫码加入建立）

## 2. 验收步骤（主流程）

### Step 1：双人绑定

1. A 端输入双方昵称与开始日期，完成绑定并展示二维码。
2. B 端进入扫码页，扫码并输入自己的昵称加入。

期望：

- A/B 均进入首页且显示相同情侣关系语义。
- B 端成功加入后，两端为同一 `pair_id`。

### Step 2：基础记录

1. A 端在时间轴新增一条文字记录（含心情、标签）。
2. A 端新增一条图片记录。
3. A 端成长页完成一次“共同任务”。
4. A 端新增一个 7 天内纪念日。

期望：

- A 首页“最近记录/成长值/纪念日提醒”即时更新。
- 成长页可看到最近完成任务列表。

### Step 3：自动同步模式

1. A/B 同时进入首页同步面板。
2. 两端点击“一键自动同步”。
3. 保持两机靠近，等待自动发现与自动同步触发。

期望：

- 同步状态出现“发现设备/已连接/同步请求已发送/接收并合并”等文案。
- B 端最终出现 A 端新增的文字、图片、成长分值、纪念日数据。
- 自动模式健康状态无持续错误累积。

### Step 4：重复同步幂等

1. 不新增任何数据，保持自动模式继续运行 1~2 分钟。

期望：

- 不会产生重复事件落库。
- 同步统计可出现重复计数，但业务展示不重复。

### Step 5：空间隔离（隐私）

1. 让第三台设备 C 使用另一个 `pair_id`（或清空后重建）并靠近 A/B。
2. 在 C 上开启可发现/发现。

期望：

- A/B 不会合并 C 的事件与文件。
- 同步状态出现“已忽略不同情侣空间同步数据/请求/文件元数据”类提示。

## 3. 验收记录模板（建议落盘）

请在测试后记录：

- 日期：`YYYY-MM-DD`
- 构建版本：`0.1.0+1`
- 设备 A 型号 / 系统版本
- 设备 B 型号 / 系统版本
- 网络状态（可离线）

结果清单：

- [ ] 双人绑定通过
- [ ] 文字记录双端同步通过
- [ ] 图片记录双端同步通过
- [ ] 成长任务与分值同步通过
- [ ] 纪念日与倒计时同步通过
- [ ] 自动模式靠近同步通过
- [ ] 重复同步幂等通过
- [ ] 跨 `pair_id` 隔离通过

截图建议：

- 绑定二维码页
- 同步面板“自动模式运行中”
- 同步后 B 端首页与时间轴
- 跨空间数据被忽略提示

## 4. 故障排查

若无法发现设备：

- 检查蓝牙/定位/WiFi 是否开启
- 检查权限是否全部授予
- 先“停止自动模式”再重新“一键自动同步”

若图片未回填：

- 等待文件传输完成提示
- 再次触发同步并确认两机未离开 Nearby 范围

若多次失败：

- 观察自动模式异常计数与最近错误
- 记录错误文案后复现并反馈
